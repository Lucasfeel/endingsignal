# Admin Console: 콘텐츠 추가 QA 체크리스트

## 사전 조건
- 관리자 계정으로 로그인되어 있어야 함
- `/admin` 또는 `/admin/contents/new` 접근 가능해야 함

## 시나리오
1. Admin Console 상단 메뉴에 `콘텐츠 추가` 탭이 보인다.
2. `콘텐츠 추가` 진입 시 `제목`, `작품 URL(선택)`, `타입`, `소스` 입력 UI가 보인다.
3. 초기 타입 옵션에 `웹툰`, `웹소설`, `OTT`가 보인다.
4. 기본 소스 입력 행은 정확히 1개이며 마지막 행은 삭제할 수 없다.
5. 타입 미선택 상태에서는 소스 선택 입력이 비활성화되고 `소스 사전 추가` 버튼이 비활성화되어 있다.
6. 타입에서 `웹툰` 선택 시 소스 목록에 `네이버웹툰`, `카카오웹툰`이 보인다.
7. `+ 소스 입력 추가` 버튼으로 소스 입력 행을 여러 개 추가할 수 있고, 각 행의 `×` 버튼으로 개별 제거가 가능하다(최소 1개 유지).
8. 타입 `+` 버튼으로 새 타입을 추가하면 즉시 타입 목록에 반영되고 자동 선택된다.
9. 소스 사전 추가 버튼으로 현재 타입의 새 소스를 추가하면 즉시 소스 목록에 반영되고 현재 폼에 선택된다.
10. `제목 + 타입 + 다중 소스`를 선택해 `추가` 제출 시 성공 토스트가 표시되고 동일 `content_id`로 소스별 콘텐츠가 생성된다.
11. `작품 URL` 입력 시 `http(s)` 형식 URL만 허용되며 저장된 콘텐츠 `meta.common.content_url`에 반영된다.
12. `콘텐츠 관리` 화면 하단 `콘텐츠 추가` 섹션에서 동일한 입력/검증/다중 소스 동작이 동작한다.
13. `콘텐츠 관리` 하단 섹션에서 생성 완료 시 목록이 갱신되고 생성된 콘텐츠가 상세 조회된다.
14. 동일 타입/소스 내 중복 타입명/소스명 등록 시 사용자에게 명확한 중복 오류가 표시된다.
15. 일반 사용자(비관리자)로 신규 API 호출 시 `403 FORBIDDEN`이 반환된다.

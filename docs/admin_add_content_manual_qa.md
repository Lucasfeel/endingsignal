# Admin Console: 콘텐츠 추가 QA 체크리스트

## 사전 조건
- 관리자 계정으로 로그인되어 있어야 함
- `/admin` 또는 `/admin/contents/new` 접근 가능해야 함

## 시나리오
1. Admin Console 상단 메뉴에 `콘텐츠 추가` 탭이 보인다.
2. `콘텐츠 추가` 진입 시 `제목`, `타입`, `소스` 입력 UI가 보인다.
3. 초기 타입 옵션에 `웹툰`, `웹소설`, `OTT`가 보인다.
4. 타입 미선택 상태에서 소스 선택과 소스 `+` 버튼이 비활성화되어 있다.
5. 타입에서 `웹툰` 선택 시 소스 목록에 `네이버웹툰`, `카카오웹툰`이 보인다.
6. 타입 `+` 버튼으로 새 타입을 추가하면 즉시 타입 목록에 반영되고 자동 선택된다.
7. 소스 `+` 버튼으로 현재 타입의 새 소스를 추가하면 즉시 소스 목록에 반영되고 자동 선택된다.
8. `제목 + 타입 + 소스`를 선택해 `추가` 제출 시 성공 토스트가 표시되고 DB에 콘텐츠가 생성된다.
9. 동일 타입/소스 내 중복 타입명/소스명 등록 시 사용자에게 명확한 중복 오류가 표시된다.
10. 일반 사용자(비관리자)로 신규 API 호출 시 `403 FORBIDDEN`이 반환된다.

## 총평

서비스의 기본 골격은 분명합니다. 탭 구조가 단순하고, "완결 알림"이라는 핵심 행동도 이해하기 쉽습니다. 검색 결과 없음, 내 구독 비어 있음, 완결됨 비어 있음 같은 빈 상태 문구도 전체적으로 친절한 편입니다.

다만 실제 사용감은 "기능은 이어지지만 확신이 덜 생기는 상태"에 가깝습니다. 가장 큰 이유는 다음 네 가지입니다.

1. 홈과 탐색 카드가 추천 서비스처럼 보이지만, 실제로는 데이터 품질과 표현 방식 때문에 신뢰감이 떨어집니다.
2. 모바일 카드 그리드가 너무 촘촘하고 정보 밀도가 높아 스캔이 어렵습니다.
3. 구독, 로그인, 상세 확인 흐름이 하나의 명확한 행동 사슬로 읽히지 않고, 모달 안에서 여러 의미가 겹칩니다.
4. 데스크톱에서는 사실상 모바일 화면을 가운데 붙여 놓은 형태라 사용성이 크게 떨어집니다.

## 핵심 문제점과 개선 방향

### 1. 홈의 "추천작"이 실제로는 추천처럼 느껴지지 않습니다

관찰:
홈 첫 화면의 `추천작`은 사용자가 기대하는 "개인화 추천"이나 "에디터 추천"이 아니라, 최신 데이터가 섞인 카드 모음처럼 보입니다. 백엔드도 실제로 웹툰/소설/OTT를 최신순으로 혼합해 가져옵니다. 관련 쿼리는 [views/contents.py#L775](C:/Users/lucas/endingsignal/views/contents.py#L775) 근처, 홈 섹션 타이틀은 [static/app.js#L5675](C:/Users/lucas/endingsignal/static/app.js#L5675) 근처입니다.

영향:
첫 진입에서 서비스가 무엇을 잘해주는지 한눈에 전달되지 않습니다. 특히 OTT 카드의 메타 텍스트가 지나치게 길고 오염되어 보여서, "추천"보다 "정제되지 않은 목록"처럼 느껴집니다.

개선 방향:
- `추천작`을 유지하려면 추천 근거를 명확히 바꾸세요.
- 예시:
  - `최근 업데이트 작품`
  - `지금 화제가 되는 완결 임박작`
  - `플랫폼별 신규 감지 작품`
- 홈은 한 가지 서사를 가져가는 편이 낫습니다.
- 추천 품질이 아직 낮다면, 홈 첫 화면은 오히려 "탭 진입 허브"로 바꾸는 편이 더 낫습니다.

### 2. 카드가 너무 작고 텍스트 중심이라 스캔성이 낮습니다

관찰:
탐색 카드가 3열 고정이며, 카드 내부도 사실상 `제목 + 한 줄 메타`만 보여줍니다. 카드 생성 로직은 [static/app.js#L6383](C:/Users/lucas/endingsignal/static/app.js#L6383), 카드 영역 템플릿은 [templates/index.html#L127](C:/Users/lucas/endingsignal/templates/index.html#L127), 카드 높이와 텍스트 영역 스타일은 [static/es_theme_toss.css#L435](C:/Users/lucas/endingsignal/static/es_theme_toss.css#L435) 근처입니다.

영향:
- 모바일에서 카드 하나당 정보가 너무 적어 작품 간 구분이 어렵습니다.
- 제목이 짧은 작품은 전부 비슷한 흰 카드로 보여 시각적 리듬이 약합니다.
- 메타가 길게 오염된 콘텐츠는 오히려 신뢰를 깎습니다.

개선 방향:
- 최소한 홈과 검색 결과는 `2열` 중심으로 바꾸고, 카드 높이를 키워 대표 썸네일 또는 타입 배지를 함께 보여주세요.
- 3열은 "완결 여부를 빠르게 훑는 밀도형 목록"일 때만 제한적으로 쓰는 편이 낫습니다.
- 메타는 작가/출연진/장르를 그대로 뿌리지 말고 타입별 정규화가 필요합니다.
- 예시:
  - 웹툰: 작가 1줄
  - 웹소설: 작가 + 장르 1줄
  - OTT: 플랫폼 + 장르 또는 출시 상태 1줄

### 3. OTT와 일부 추천 카드의 메타 데이터가 과도하게 노출됩니다

관찰:
실제 화면에서 일부 OTT 카드가 위키/기사 원문처럼 긴 텍스트를 그대로 메타에 노출합니다. 이는 첫 화면과 OTT 탭 모두에서 확인됐습니다. 카드 메타는 [static/app.js#L6413](C:/Users/lucas/endingsignal/static/app.js#L6413) 근처에서 `authors` 값을 그대로 사용합니다.

영향:
- 사용자는 카드의 정보 구조를 신뢰하기 어렵습니다.
- 콘텐츠 품질 문제가 UI 문제로 보이게 됩니다.
- "완결 알림 서비스"보다 "크롤링 결과 뷰어"처럼 보일 수 있습니다.

개선 방향:
- 노출 전용 `display_meta`를 타입별로 강하게 제한하세요.
- 길이 제한만 두지 말고, 필드 자체를 목적에 맞게 바꿔야 합니다.
- 홈/탐색 카드에서 자유 텍스트 계열 메타는 제거하고, 구조화된 짧은 필드만 쓰는 게 안전합니다.

### 4. 구독 모달은 핵심 행동이 있지만, 상세 확인 흐름은 불명확합니다

관찰:
카드를 누르면 별도 상세 페이지가 아니라 구독 모달이 열립니다. 모달 자체는 목적에 맞지만, 실제 원작/플랫폼 링크는 별도 CTA가 아니라 제목 텍스트에만 걸려 있습니다. 모달 템플릿은 [templates/index.html#L153](C:/Users/lucas/endingsignal/templates/index.html#L153), 링크 삽입 로직은 [static/app.js#L6576](C:/Users/lucas/endingsignal/static/app.js#L6576) 근처입니다.

영향:
- 사용자는 "카드를 눌렀는데 왜 상세 화면이 아니라 작은 모달이 뜨지?"라는 인상을 받을 수 있습니다.
- 작품 확인과 구독이 한 화면에서 섞이지만, 각 행동의 우선순위가 분명하지 않습니다.
- 제목이 링크라는 사실을 모르면 외부 상세를 열 수 없다고 느낄 수 있습니다.

개선 방향:
- 모달에 명시적인 보조 CTA를 넣는 편이 좋습니다.
- 예시:
  - `작품 보러 가기`
  - `완결 알림 구독`
- 닫기 버튼도 추가하는 편이 안전합니다. 현재는 오버레이 클릭으로 닫을 수 있지만, 시각적으로는 닫기 수단이 명확하지 않습니다.

### 5. 인증 흐름은 동작하지만, 피드백 방식이 일관되지 않습니다

관찰:
- 구독 시 비로그인 상태에서는 토스트(`로그인이 필요합니다.`)와 로그인 모달이 동시에 뜹니다. 관련 코드는 [static/app.js#L2343](C:/Users/lucas/endingsignal/static/app.js#L2343) 근처입니다.
- 반면 `내 구독` 탭에서는 별도 빈 상태와 `로그인하기` CTA를 보여줍니다. 관련 코드는 [static/app.js#L5990](C:/Users/lucas/endingsignal/static/app.js#L5990) 근처입니다.
- 잘못된 로그인 시도에서는 어떤 경우 `세션이 만료되었습니다. 다시 로그인해주세요.` 토스트가 먼저 보이고, 이어서 `이메일 또는 비밀번호가 올바르지 않습니다.`가 나타났습니다. 세션 관련 토스트는 [static/app.js#L2559](C:/Users/lucas/endingsignal/static/app.js#L2559) 근처입니다.

영향:
- 같은 "로그인이 필요한 상황"인데도 패턴이 다르게 느껴집니다.
- 잘못된 로그인 앞에 세션 만료 토스트가 먼저 뜨면 사용자가 원인을 잘못 해석할 수 있습니다.

개선 방향:
- 인증 요구 패턴을 2개만 유지하는 것이 좋습니다.
- 권장:
  - 페이지 단위 접근 제한: 빈 상태 + CTA
  - 액션 단위 접근 제한: 모달 진입 또는 바텀시트, 토스트는 보조로만 사용
- 세션 만료 토스트는 실제 만료 상황에서만 보여주고, 신규 비로그인 사용자의 첫 액션에서는 숨기는 편이 낫습니다.

### 6. 검색 첫 진입 상태가 "깨진 화면"처럼 느껴질 수 있습니다

영향:

- 첫 검색 전의 대기 화면이 서비스 신뢰를 깎습니다.

개선 방향:
- 검색 첫 진입 화면은 실패 상태가 아니라 안내 상태처럼 보여야 합니다.
- `최근 검색어 없음`과 `인기 검색어` 또는 `최근 업데이트 작품`을 안정적으로 보여주는 쪽이 좋습니다.
- 적어도 실패 문구 대신 기본 추천 셋을 별도 API로 가져오는 편이 낫습니다.

4. 검색 유휴 상태에서 `추천 작품을 불러오지 못했습니다.` 제거

### P1. 바로 체감되는 품질 개선

1. 모바일 카드 그리드를 2열 중심으로 재설계
2. 하단 탭 라벨 크기와 필터 가독성 개선
3. 인증 요구 방식 통일
4. 카운트 문구 통일

### P2. 다음 단계 개선

1. 데스크톱/태블릿 브레이크포인트 추가

## 결론

지금 서비스는 "핵심 기능은 살아 있는데, 첫인상과 탐색 품질이 기능 가치를 충분히 전달하지 못하는 상태"입니다.

가장 좋은 점은 기능 흐름이 이미 이어진다는 것입니다. 회원가입, 로그인, 구독, 내 구독 확인까지는 실제로 동작합니다. 그래서 지금 필요한 것은 전면 재구축보다도, `첫 화면의 신뢰감`, `카드 정보 설계`, `모달 액션 명확화`, `검색 유휴 상태 정리` 쪽의 집중 개선입니다.

이 네 가지만 바로잡아도 서비스 완성도는 한 단계 크게 올라갈 가능성이 높습니다.
